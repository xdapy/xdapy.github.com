I/O formats¶

xdapy can read and write data in both XML and JSON formats. A specification of these formats is given here.

XML¶

For XML output to work, we first have to initialise the XML serialiser. We supply our current object mapper as a parameter:

from xdapy.io import XmlIO
xmlio = XmlIO(mapper)

This will automatically register all known object mappings.

If we have data stored inside our database, we can now access the XML DOM tree through:

xmltree = xmlio.write()

And print it:

from xml.etree import ElementTree as ET
ET.tostring(xmltree)

This may give us something like the following:

<xdapy>
    <types>
        <entity type="Trial">
            <parameter name="rt" type="string"/>
            <parameter name="valid" type="boolean"/>
            <parameter name="response" type="string"/>
        </entity>
        <entity type="Experiment">
            <parameter name="project" type="string"/>
            <parameter name="experimenter" type="string"/>
        </entity>
        <entity type="Session">
            <parameter name="date" type="date"/>
        </entity>
        <entity type="Observer">
            <parameter name="age" type="integer"/>
            <parameter name="handedness" type="string"/>
            <parameter name="name" type="string"/>
        </entity>
    </types>
    <objects>
        <entity id="1" type="Observer" uuid="e4a7918b-8317-427d-800e-1a221df38db2">
            <parameter name="age" value="200"/>
        </entity>
        <entity id="2" type="Experiment" uuid="663076b4-f35d-40f2-8cc9-b06b2fb27e48">
            <entity id="3" type="Session" uuid="115bc5fe-da2c-458a-85d9-c8786762801f">
                <entity id="4" type="Trial" uuid="c26ebe68-c6d2-4a9a-bb73-2d783737b992">
                    <parameter name="valid" value="FALSE"/>
                </entity>
                <entity id="5" type="Trial" uuid="57019f27-8f43-445a-b41e-bfebe644f9d8">
                    <parameter name="valid" value="TRUE"/>
                </entity>
            </entity>
        </entity>
    </objects>
    <relations>
        <context from="id:2" name="observed by" to="id:1"/>
    </relations>
</xdapy>

And a more complicated example:

<xdapy>
    <types>
        <entity type="Observer">
            <parameter name="age" type="integer" />
            <parameter name="handedness" type="string" />
            <parameter name="name" type="string" />
        </entity>
    </types>
    <objects>
        <entity id="1" type="Observer" uuid="e4a7918b-8317-427d-800e-1a221df38db2">
            <parameter name="age" value="200" />
        </entity>
    </objects>
    <relations />
</xdapy>

References between several entity objects are expressed either by using the id variable or the uuid identifier. The id is seen as a local identifier: It is not retained during import or export of objects. Each id is only unique for the medium it is defined in. E.g. for several input files of XML data, there may be several objects with id="92". All references with a specifier "id:92" can of course only be resolved inside that file.

For global references, the object mapper introduces a UUID. This identifier may be used for stable referencing across several files.

In general, the object mapper will take care of generating these UUID numbers. When creating a XML or JSON file for input, it is sufficient (and advisable) to only use id references and leave out uuid values.

JSON¶

Alternatively, there is also a simple JSON-compatible format available.

{
    "objects": [
        {
            "id": 1, 
            "parameters": [
                {
                    "age": 200
                }
            ], 
            "type": "Observer", 
            "uuid": "e4a7918b-8317-427d-800e-1a221df38db2"
        }, 
        {
            "children": [
                {
                    "children": [
                        {
                            "id": 4, 
                            "parameters": [
                                {
                                    "valid": false
                                }
                            ], 
                            "type": "Trial", 
                            "uuid": "c26ebe68-c6d2-4a9a-bb73-2d783737b992"
                        }, 
                        {
                            "id": 5, 
                            "parameters": [
                                {
                                    "valid": true
                                }
                            ], 
                            "type": "Trial", 
                            "uuid": "57019f27-8f43-445a-b41e-bfebe644f9d8"
                        }
                    ], 
                    "id": 3, 
                    "type": "Session", 
                    "uuid": "115bc5fe-da2c-458a-85d9-c8786762801f"
                }
            ], 
            "id": 2, 
            "type": "Experiment", 
            "uuid": "663076b4-f35d-40f2-8cc9-b06b2fb27e48"
        }
    ], 
    "relations": [
        {
            "from": "id:2", 
            "name": "observed by", 
            "relation": "context", 
            "to": "id:1"
        }
    ], 
    "types": [
        {
            "parameters": [
                {
                    "name": "rt", 
                    "type": "string"
                }, 
                {
                    "name": "valid", 
                    "type": "boolean"
                }, 
                {
                    "name": "response", 
                    "type": "string"
                }
            ], 
            "type": "Trial"
        }, 
        {
            "parameters": [
                {
                    "name": "project", 
                    "type": "string"
                }, 
                {
                    "name": "experimenter", 
                    "type": "string"
                }
            ], 
            "type": "Experiment"
        }, 
        {
            "parameters": [
                {
                    "name": "date", 
                    "type": "date"
                }
            ], 
            "type": "Session"
        }, 
        {
            "parameters": [
                {
                    "name": "age", 
                    "type": "integer"
                }, 
                {
                    "name": "handedness", 
                    "type": "string"
                }, 
                {
                    "name": "name", 
                    "type": "string"
                }
            ], 
            "type": "Observer"
        }
    ]
}

The JSON representation is compressed in the sense that the parameters section of an entity is represented by a single dictionary (JSON object). In JSON we also use the directly corresponging representation of simple data types. E.g. we simply use 1.0 instead of "1.0" for a floating point number.
